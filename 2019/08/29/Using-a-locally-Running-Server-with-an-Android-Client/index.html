Developing on a physical Android client with a local server

Mike Kamau's Blog
2019-08-29 (Updated: 2019-08-29)
android

When developing Android clients, many a time I will be running local servers on my machine. Sometimes, I also need to run the client on a physical Android device.

Suppose the local server is running on localhost:8085. To access this on an Android emulator is straightforward since that address maps to http://10.0.2.2:8085. However, for a physical device, using 10.0.2.2 will not necessarily work.

To get it to work, the approach I use to first connect both my development machine and phone to the same network, usually Wifi. I then check the IP address of my machine by running the ifconfig command to list all active network interfaces. Normally, the correct address will be under the en0 interface. You can of course check for the IP address under Network setting of the operating system. If say the IP address is 192.168.0.37, then using it on the physical Android device will work.

This is not convenient since I have to keep checking and changing the IP addresses every time I change networks.
